- Summary: File over app emphasizes the importance of creating digital artifacts as files that are easily controlled and stored in accessible formats. It urges tool makers to prioritize user ownership of data, highlighting that files have lasting significance compared to the transient nature of applications. Drawing parallels to ancient hieroglyphs, the philosophy underscores the enduring value of tangible artifacts and stresses the need for durable digital files that can be retrieved across time. The message resonates with the idea that ensuring the longevity of one's work involves prioritizing the accessibility and preservation of files over the tools used to create them.

## Highlights

- *File over app* is a philosophy: if you want to create digital artifacts that last, they must be files you can control, in formats that are easy to retrieve and read. Use tools that give you this freedom. ([View Highlight](https://read.readwise.io/read/01ht3fatff5q049d1c59hjnm6g))
- You should want the files you create to be durable, not only for posterity, but also for your future self. You never know when you might want to go back to something you created years or decades ago. Don’t lock your data into a format you can’t retrieve.
	These days I write using an app I help make called [Obsidian](https://stephango.com/obsidian), but it’s a delusion to think it will last forever. The app will eventually become obsolete. It’s the plain text files I create that are designed to last. Who knows if anyone will want to read them besides me, but *future me* is enough of an audience to make it worthwhile.
