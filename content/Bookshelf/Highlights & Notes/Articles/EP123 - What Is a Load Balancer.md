- A load balancer is a device or software application that distributes network or application traffic across multiple servers. ([View Highlight](https://read.readwise.io/read/01j54e4s0n3ys792ej8j2km895))

- What Does a Load Balancer Do?
	1. Distributes Traffic
	2. Ensures Availability and Reliability
	3. Improves Performance
	4. Scales Applications ([View Highlight](https://read.readwise.io/read/01j54e5px8mcmran7seaxctdhk))
- Types of Load Balancers ([View Highlight](https://read.readwise.io/read/01j54e5szssnqcm9dj33kkxcbk))
- 1. Hardware Load Balancers: These are physical devices designed to distribute traffic across servers. ([View Highlight](https://read.readwise.io/read/01j54e5yqwb6z12g57wdq1md4b))
- 2. Software Load Balancers: These are applications that can be installed on standard hardware or virtual machines. ([View Highlight](https://read.readwise.io/read/01j54e62b8h4a6jzt7kt3asp0e))
- 3. Cloud-based Load Balancers: Provided by cloud service providers, these load balancers are integrated into the cloud infrastructure. Examples include AWS Elastic Load Balancer, Google Cloud Load Balancing, and Azure Load Balancer. ([View Highlight](https://read.readwise.io/read/01j54e6c5qcnb86xd8pkzffrx5))
- 4. Layer 4 Load Balancers (Transport Layer): Operate at the transport layer (OSI Layer 4) and make forwarding decisions based on IP address and TCP/UDP ports. ([View Highlight](https://read.readwise.io/read/01j54e6jtqp9h0fc2nwb0yywwr))
- 5. Layer 7 Load Balancers (Application Layer): Operate at the application layer (OSI Layer 7) . ([View Highlight](https://read.readwise.io/read/01j54e6xns1jtxtvs12fx33p76))
- 6. Global Server Load Balancing (GSLB): Distributes traffic across multiple geographical locations to improve redundancy and performance on a global scale. ([View Highlight](https://read.readwise.io/read/01j54e74dwd6k06c65zjqefb4w))
