# Maintaining Software is My Goal

- John Cricket #people/authors/john-cricket
- [source](https://www.linkedin.com/posts/johncrickett_youll-never-be-a-great-software-engineer-activity-7114942168384839680-3G-I)

> [!quote] You’ll never be a great software engineer if you’ve never maintained code.

Why?

Because building greenfield software doesn’t take long. Not compared to how long that software will be used.

It’s easy to build greenfield software and then disappear off to a new role.

The hard bit about software development isn’t building something new. It’s supporting something that already exists. 💯

It’s adding new features to it, whilst not breaking the existing features.

It’s working around, living with and resolving the technical debt.

It’s re-designing the system when it turns out the real-world interacts with it in ways you didn’t expect, whilst you have to keep the system going.

It’s having the experience of:
- you build it → you run it → you own it.

Without the lessons you learn from those experiences you’ll never have the complete experience of being a software engineer. Without that experience you can never be a great software engineer.

The great news is, very few of us can escape the experience of maintaining legacy code.

I encourage you to embrace it as a growth opportunity, the lessons you learn with help you grow from being a good software engineer to a great software engineer.
